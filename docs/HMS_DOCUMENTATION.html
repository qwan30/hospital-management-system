Architecture Decisions

Key Design Choices & Rationale

4 Architecture Decision Records documenting intentional trade-offs.

ADR-001
Modular Monolith over Microservices
Healthcare has tightly-coupled workflows (appointment → queue → ehr → prescription → billing). A modular monolith preserves transactional consistency while DDD bounded contexts prevent coupling. Microservices would add distributed transaction complexity and operational overhead without benefit at this scale.
Read full ADR →
ADR-002
Repository Interfaces in Domain Layer
Placing repository contracts in the domain enforces the Dependency Inversion Principle — domain owns the contract, infrastructure implements it. Prevents infrastructure concerns from leaking into business logic.
Read full ADR →
ADR-003
JWT + httpOnly Refresh Cookies
Short-lived JWT access tokens (15min) + httpOnly refresh cookies (7-day rotation). httpOnly prevents XSS token theft. Stateless JWTs eliminate server-side session storage.
Read full ADR →
ADR-004
AES-GCM PHI Encryption with SHA-256 Indexing
Patient identifiers (CCCD/CMND) encrypted at rest with AES-GCM, indexed by SHA-256 hash for lookup without decryption. Plaintext never stored. Satisfies healthcare PHI compliance.
Read full ADR →

Quick Start

Get Running in Under a Minute

Prerequisites: Java 17+, Node.js 22+, Docker Desktop.

1

Start PostgreSQL

docker compose -f infra/docker-compose.yml up -d postgres
2

Backend

cp .env.example .env
cd backend
mvn spring-boot:run -f start/pom.xml

:8081/actuator/health

3

Frontend

cd frontend
npm install && npm run dev

localhost:3000
