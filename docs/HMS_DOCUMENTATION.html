Project Summary

A complete clinical operations platform

From public appointment booking through clinical consultation, pharmacy dispensing, and billing — every clinical workflow is modeled as a bounded context in the backend and surfaced through role-specific frontend routes.

Public Booking & Discovery

Department directory, doctor profiles, news, and a multi-step booking wizard with slot-locking transactions. Patient identifiers encrypted with AES-GCM and indexed by SHA-256 hash.

Clinical Intake & Queue

Full queue lifecycle — check-in, room assignment, consultation start, completion, and skip. Nurse vital-sign capture integrated with the clinical record.

EHR & Prescriptions

Electronic health records with digital prescriptions. Automated PDF generation via Apache PDFBox. Follow-up scheduling and Gmail-based reminder delivery.

Pharmacy & Inventory

Lot/movement-tracked drug inventory with dispense traceability linked to prescription IDs. Stock alerts and audit-trail for every movement.

Billing & Revenue

Auto-generated invoices from service pricing tables. Payment recording, void support, and daily/monthly revenue reports for accounting.

Patient Portal

Self-service access to appointments, lab results, messages, and profile. JWT + httpOnly refresh-cookie authentication. Claim-based initial access.

System Flows

End-to-end clinical workflows

Seven primary flow families connect roles, frontend routes, and backend API families.

1. Public Discovery & Booking

Guest

Home page, department/doctor directory, news, and multi-step booking wizard with slot-lock transactions.

/content/home · /departments · /doctors · /appointments · /chatbot/messages

2. Staff Authentication

All Staff Roles

JWT login with httpOnly refresh cookie. Access token stored in session storage. Route guards on frontend.

/auth/login · /auth/refresh · /auth/logout

3. Queue Operations

Admin · Nurse · Receptionist

Full queue lifecycle: check-in, room assignment, consultation start, completion, skip. Nurse intake with vital signs.

/queue/today · /appointments/{id}/checkin · /queue/{id}/call · /queue/{id}/assign-room · /queue/{id}/start-consultation · /queue/{id}/complete · /queue/{id}/skip

4. Doctor Clinical Work

Admin · Doctor · Nurse

Medical records, prescriptions with PDF generation, follow-up scheduling, lab results, vital signs. Role-gated at backend.

/me/schedule · /patient-records · /medical-records · /appointments/{id}/status · /appointments/{id}/follow-up · /lab-results · /vital-signs

5. Patient Portal

Patient

Claim-based initial access, overview dashboard, appointment list, lab results, messages (read-only), and profile.

/patient-auth/claim · /patient-auth/login · /patient-portal/overview · /patient-portal/appointments · /patient-portal/lab-results

6. Inventory & Pharmacy

Admin · Pharmacist

Lot/movement-tracked inventory, dispense with prescription traceability, stock alerts.

/inventory/items · /inventory/lots · /inventory/movements · /inventory/dispense · /inventory/alerts

7. Finance & Admin

Admin · Accountant

Invoices, payments, void, pricing management, daily/monthly revenue reports, audit logs, user/room management.

/invoices · /pricing · /reports/revenue/* · /admin/users · /admin/departments · /admin/audit-logs · /admin/stats

Current Limitations

  • Patient portal message send/reply not implemented (read-only).
  • Patient self-cancel and reschedule APIs not implemented.
  • Separate live nurse room-board API not implemented; queue assign-room is the active room workflow.
  • External payment-gateway integration is out of scope.
  • Selected frontend routes exist but are not fully backend-integrated.

CI/CD & DevOps

Automated pipelines with observability

Four GitHub Actions workflows cover build, deploy, rollback, and security scanning.

CI Pipeline

Builds Java backend (Maven), runs integration tests with Testcontainers, lints frontend, runs Vitest and Playwright suites, builds Docker images, and pushes to GHCR.

CD Pipeline

Deploys to VPS via Docker Compose. Pulls latest images from GHCR, runs database migrations, and restarts services.

Rollback

Automated rollback workflow for reverting to a previous deployment state if a release fails health checks.

Security Scan

Secret scanning (high-confidence), dependency vulnerability audit, and SAST checks on every push.

Getting Started

Run the system locally

Prerequisites: Java 17+, Node.js 22+, Docker Desktop. Copy .env.example to .env and fill in required secrets.

Docker Compose (Full Stack)

docker compose up -d
# PostgreSQL + Backend + Frontend
# http://localhost:3000

Observability: docker compose -f docker-compose.yml -f docker-compose.observability.yml up -d

Backend (Development)

.\backend\run.ps1
# or manual:
cd backend
mvn install -DskipTests
mvn spring-boot:run -f start/pom.xml
# http://localhost:8081

Frontend (Development)

cd web
npm install
npm run dev
# http://localhost:3000

Verification Commands

# Backend health
curl http://localhost:8081/actuator/health

# Backend tests
cd backend && mvn verify

# Frontend tests
cd web
npm run test:unit
npm run test:e2e:ui
