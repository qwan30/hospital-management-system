# Hospital Management System — Claude Code Instructions

This project uses **Everything Claude Code (ECC)** — a comprehensive AI agent harness with 47 specialized agents, 181 skills, hooks, rules, and MCP configurations.

## ECC Installation Location

All ECC components are installed in the `.agents/` directory:

```
.agents/
  ├── agents/      # 47 specialized subagents
  ├── skills/      # 181 workflow skills and domain knowledge
  ├── rules/       # Language-specific coding guidelines
  ├── hooks/       # Trigger-based automation (hooks.json)
  ├── commands/    # Legacy slash-command shims
  ├── mcp-configs/ # MCP server configurations
  ├── scripts/     # Cross-platform utilities
  └── tests/       # Test suite
```

## Project-Specific Configuration

The following root-level files configure Claude Code for this project:

- **AGENTS.md** — Agent definitions and orchestration rules
- **CLAUDE.md** — Claude Code-specific configuration with GitNexus
- **AGENTS.md** — Core principles, security guidelines, testing requirements

## Quick Start

### Using ECC Skills
Skills provide the primary workflow surface. Access via:
- `/ecc:plan` — Implementation planning
- `/ecc:tdd` — Test-driven development
- `/ecc:code-review` — Code quality review
- `/ecc:security` — Security-first review
- And 178+ more skills in `.agents/skills/`

### Using Specialized Agents
For complex work, specialized agents handle domain tasks autonomously:
- **planner** — Break complex features into phases
- **tdd-guide** — Write tests first, then implementation
- **code-reviewer** — Enforce quality standards
- **security-reviewer** — Scan for vulnerabilities
- **database-reviewer** — PostgreSQL/Supabase optimization
- **architect** — System design and scalability

### Using GitNexus

This project is indexed by GitNexus for code intelligence:

# Check index freshness
npx gitnexus status

npx gitnexus analyze

# Full reindex with embeddings
npx gitnexus analyze --embeddings

Before editing any symbol:
```

```

## Core Development Principles

1. **Test-Driven Development** — 80%+ test coverage required
2. **Security-First** — Validate all inputs, never hardcode secrets
3. **Agent-First** — Delegate to specialized agents for domain work
4. **Immutability** — Always create new objects, never mutate
5. **Plan Before Execute** — Use planner for complex features

## Testing Requirements

**Minimum coverage: 80%**

All projects require:
- **Unit tests** — Individual functions, utilities, components
- **Integration tests** — API endpoints, database operations
- **E2E tests** — Critical user flows

## Hooks and Automation

ECC includes trigger-based automations configured in `.agents/hooks/hooks.json`:

- **SessionStart** — Initialize session state
- **PreToolUse** — Validate before tool execution
- **PostToolUse** — Post-process and verify results
- **Stop** — Clean up on session end

## Adding Project-Specific Skills

To add custom project skills:

1. Create `.agents/skills/project-name/SKILL.md`
2. Follow the skill format in existing skills
3. Reference in agent tasks with `/ecc:project-name`
